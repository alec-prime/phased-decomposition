# WP-01-02: Description Trigger-Rate Evals

## Work Package Scope

Author 20 trigger-rate eval queries for description optimization — mix of should-trigger and should-not-trigger. Output is a JSON array conforming to skill-creator's trigger eval format. Queries must be realistic, concrete, edge-case-heavy.

## Inputs

- `SPEC.md § SKILL.md Body Structure → Frontmatter` — description constraints (third person, pushy, under 1024 chars, signals project-shaped work)
- `ARCHITECTURE.md § Skill description` — description authoring constraints
- `skill-creator SKILL.md § Description Optimization` — trigger eval format and query quality guidance

## Acceptance Criteria

- Exactly 20 queries: 10 should-trigger, 10 should-not-trigger
- Should-trigger queries cover different phases and project types
- Should-not-trigger queries are near-misses, not obviously irrelevant
- Queries are concrete with realistic detail per skill-creator guidance (file paths, personal context, specific tools)
- Format: `[{"query": "...", "should_trigger": true/false}]`
