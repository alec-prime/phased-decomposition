# WP-08-01: Cross-Reference Conventions Pass

## Work Package Scope

Final-pass review and correction of cross-reference conventions across all produced skill files. Checks first-instance links, frontmatter lineage, and stub sections per `SPEC.md § Cross-Reference Conventions`.

## Inputs

- `SPEC.md § Cross-Reference Conventions` — two mechanisms (frontmatter lineage + first-instance links), aliasing examples, stub section pattern, consistency rule
- All produced skill files (SKILL.md, all reference files, all asset templates)

## Acceptance Criteria

- Every artifact named in a file's `descends_from` appears at least once in the body as a first-instance wikilink
- First bare-prose occurrence of another artifact's name is wrapped in an Obsidian wikilink aliased to preserve surface grammar
- Subsequent mentions of the same artifact are unlinked
- Self-references are excluded
- Parent-child stub sections use the committed format
- Code-span filename references are not linked; first bare-prose occurrence gets the link
- No body link points to an artifact absent from `descends_from` without explanation
