# WP-01-01: Core Behavioral Evals

## Work Package Scope

Author eval entries for four behavioral categories: phase routing, reference-file discovery, interrogator-before-production, and kernel-surfacing protocol. Output is entries in `evals/evals.json` conforming to skill-creator's schema. Each category gets positive cases (prompts that should produce the behavior) and negative cases (prompts exposing failure modes from `SPEC.md § Interrogator` and `§ Scribe`).

## Inputs

- `SPEC.md § Interrogator` — failure examples for kernel-surfacing protocol negative cases
- `SPEC.md § Scribe` — failure examples for scribe voice negative cases
- `SPEC.md § Per-Phase Role Application` — which roles run in which phases, for phase-routing cases
- `ARCHITECTURE.md § Eval categories` — the five categories, their descriptions
- `skill-creator/references/schemas.md` — `evals.json` schema (fields: id, prompt, expected_output, files, expectations)

## Acceptance Criteria

- At least 2 positive and 1 negative case per behavioral category
- Kernel-surfacing protocol cases include negative cases for: proposal-with-checkback, kernel slipped into summary, stopping at mechanism, production without interrogation
- All entries conform to skill-creator's `evals.json` schema
- Prompts are realistic user utterances, not abstract descriptions
